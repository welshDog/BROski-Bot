# Security Policy

## Reporting a Vulnerability

**DO NOT** open a public GitHub issue to report security vulnerabilities.

Instead, please report security issues privately to **welshDog** via GitHub's [Private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories).

### What to Include
- Description of the vulnerability
- Steps to reproduce (if applicable)
- Potential impact
- Suggested fix (if you have one)

### Response Time
We will acknowledge your report within **48 hours** and provide updates every **7 days**.

## Security Best Practices

### For Users
- Always use the latest version from the `main` branch
- Enable 2FA on your GitHub account
- Never commit secrets, API keys, or credentials to any branch
- Report suspicious activity immediately

### For Contributors
- Use `AGPL-3.0` license for any derivative works
- Sign your commits with GPG when possible
- Follow our `CODE_OF_CONDUCT.md`
- Review `CONTRIBUTING.md` before submitting PRs

## Supported Versions

| Version | Status |
|---------|--------|
| Latest (main) | ✅ Actively maintained |
| Previous stable | ⚠️ Security patches only |
| Older | ❌ No longer supported |

## Dependencies & Updates

- Dependabot scans for vulnerable dependencies daily
- Critical security updates are prioritized immediately
- Subscribe to security alerts in this repo's `Watch` settings

## Thank You

Security researchers who responsibly disclose vulnerabilities help make this project safer for everyone. We appreciate your help! 🙏
